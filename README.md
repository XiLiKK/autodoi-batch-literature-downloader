# 📚 DOI 批量文献自动下载助手 (Anti-Bot & Sci-Hub 终极版)

这是一个为研究人员量身定制的批量文献下载 CLI 工具。它集成了最前沿的反爬虫模拟技术（Playwright-Stealth）、多重下载通道（官方直链、浏览器按钮抓取）、全字段重命名，以及在官方受限时的 **Sci-Hub 无缝切换兜底** 功能。

---

## ✨ 核心特性

1. **极致净爽的重命名**：自动将杂乱的文件名规范为 `年份_期刊缩写_第一作者姓_论文名.pdf`（如 `2019_JMPS_Lo_A phase-field model.pdf`）。
2. **免配邮箱也能跑**：移除繁琐 API 强制限制，只需一键启动。
3. **隐形浏览器脱壳技术**：内置 Stealth 模式，极大降低触发 Elsevier、Wiley 等高门槛数据库人机验证（CAPTCHA）的概率。
4. **终极防线兜底**：当没有高校内网权限导致触发付费墙时，系统会自动轮询调用 `.se`/`.ru`/`.st` 等俄区服务器，免费拽回文献。
5. **增量下载不重复**：自带轻量级数据库，断点续传，不小心关掉电脑再跑一次，绝对不会把已下载的文章重新下一次。
6. **人工补漏队列**：对于极其新、无权限且 Sci-Hub 未收录的“钉子户”，程序会妥善归置在“待办列表”里，让你一键自动打开浏览器逐个检查。

## 🛠️ 第一步：环境配置（只需一次）

在你要使用该工具的电脑上，打开 **命令行终端 (Powershell / CMD / Terminal)**，确认你处于本项目的根目录下（`d:\Antigravity_kk\Project_kk\20260403-test`）。

1. **安装依赖的 Python 库库**:
   ```bash
   pip install httpx rich bs4 pyyaml playwright playwright-stealth
   ```
2. **下载自动化浏览器内核 (Playwright驱动)**:
   ```bash
   playwright install chromium
   ```

*(注：如果你已经在这个电脑上跑成功过，这段就可以永远跳过了)*

---

## 📑 第二步：放入你的 DOI

打开当前文件夹里的 `input/papers.csv` 文件。这是你的投喂口。
你只需要保证这个表里面有一列名叫 **`DOI`**。

比如用 Excel 或记事本打开，把内容改成类似这样：
```csv
DOI
10.1016/j.jmps.2019.103684
10.1007/s10704-022-00628-0
10.1016/j.ijfatigue.2023.107558
```
保存并关闭。

## 🚀 第三步：日常运行命令

以后你离开 AI 助手自己独立使用时，所有的魔法都在这三个核心指令里中（请在文件夹根目录下执行）：

### 1. 自动轰炸下载（最常用！）
这是你每天用得最多的一句：
```bash
3
```
当你敲下回车，就可以端起咖啡离开了。它会自动经历重重关卡，并把 PDF 塞进 `output/downloads` 文件夹里。

### 2. 人工检修“钉子户”
等上面的命令全跑完了之后，肯定有一些（比如全网最新的 2025 年的且非开源的）硬骨头。跑下面这句命令：
```bash
python -m paper_downloader manual
```
程序立刻会接管你的键盘：它会逐一帮你把那些没下载成功的原网页在浏览器里打开，你如果有号或者是找同学借的高校 VPN，就可以直接在页面上点下载。下载完点击回车，它会自动给你开下一个网页。

### 3. 生成科研小报告
想知道到底捞了哪些文献下来吗？
```bash
python -m paper_downloader stats
```
这会在控制台打出极其漂亮的彩色统计表格。

## ☁️ 第四步：GitHub Actions 自动化部署 (云端全自动)

本版本支持直接在 GitHub 环境下利用 GitHub Actions 实现“打通自动下载 -> 打包文献” 的云端工作流。你不需要在本地保留任何运行环境。

### **使用流程**：
1. **Fork / Clone 此仓库** 到你自己的 GitHub。
2. **设置环境变量（安全存储）**：
   在你的仓库中点击 **Settings** -> **Secrets and variables** -> **Actions**，点击 `New repository secret`，添加如下参数：
   - Name: `PAPER_DOWNLOADER_EMAIL`
   - Secret: `你的邮箱地址` (进入 Crossref 礼貌池的邮箱，具体参考 `.env.example` 文件中的声明)
   
   ⚠️ ***注意：安全声明***：本版本的所有代码逻辑**绝对不会**自动上传或者暴露个人 API Keys 或邮箱等敏感信息到前端或公共网页。请务必将真实的凭据信息记录在不提交的 `.env` 文件或 GitHub Secrets 库中。

3. **触发工作流**：
   - **自动触发**：只要你向仓库的 `input/papers.csv` 文件 Push 新的 DOI 列表，系统立刻在云端运行。
   - **手动触发**：前往 GitHub **Actions** 选项卡，选择 `Auto Download Papers` 并点击 `Run workflow`。

4. **提取打包文献资源包**：
   流线执行完成后，在 Actions 的“Artifacts”列表点击 `Downloaded-Papers`，下载后的 PDF 资源会自动作为压缩包提供给你。
